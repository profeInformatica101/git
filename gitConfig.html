Mostrar configuración completa con git config -l

Visualizar configuración global y local

El comando git config -l (o --list) muestra todas las configuraciones activas, combinando los niveles:

  • Global: almacenadas en ~/.gitconfig
  • Local: del repositorio actual (.git/config)
  • Sistema: configuraciones de instalación
git config -l

Si deseas solo las opciones globales: git config --global -l.

Secciones de configuración

user.*

¿Qué hace?

  • user.name: Nombre que aparecerá como autor de los commits.
  • user.email: Correo electrónico asociado al autor.
[user]
    name = Pablo
    email = jdommay863@g.educaand.es
safe.directory

¿Qué hace?

Marca el directorio como “seguro”, evitando la advertencia detected dubious ownership in repository.

[safe]
    directory = /home/kronos/Documentos/endes_git
alias.tree

¿Qué hace?

Crea un atajo de comando. Con git tree obtienes el historial en formato gráfico.

[alias]
    tree = log --graph --decorate --all --oneline
credential.helper

¿Qué hace?

Almacena temporalmente tus credenciales en memoria (15 min por defecto), evitando reintroducir usuario/token.

[credential]
    helper = cache
core.*

¿Qué hace?

  • repositoryformatversion=0: Versión del formato interno de Git.
  • filemode=true: Git respeta los permisos (ejecutable, lectura…).
  • bare=false: Repositorio no “bare” → tiene árbol de trabajo.
  • logallrefupdates=true: Registra movimientos de referencias.
[core]
    repositoryformatversion = 0
    filemode = true
    bare = false
    logallrefupdates = true
remote.origin.*

¿Qué hace?

  • url: Dirección SSH del repositorio en GitHub.
  • fetch: Al hacer git fetch, trae todas las ramas como origin/<nombre>.
[remote "origin"]
    url = git@github.com:profeInformatica101/miapp.git
    fetch = +refs/heads/*:refs/remotes/origin/*
branch.*

¿Qué hace?

Define cómo cada rama local se sincroniza con su contraparte remota.

[branch "main"]
    remote = origin
    merge  = refs/heads/main

[branch "curl_SwingMavenBase"]
    remote = origin
    merge  = refs/heads/curl_SwingMavenBase

[branch "feature-date"]
    remote = origin
    merge  = refs/heads/feature-date
... (y así sucesivamente)

Al hacer git pull en estas ramas, Git obtiene los cambios desde <remote>/<merge>.

Cómo mostrarlo en GitHub (Pages + README)

Publicar con GitHub Pages
  1. Crea la carpeta docs/ en tu repo y guarda este archivo como docs/index.html.
  2. Sube y haz commit/push.
  3. En tu repo: Settings → Pages.
  4. En “Branch”, elige main y carpeta /docs. Guarda.
  5. Espera unos segundos: tu web quedará disponible en https://<usuario>.github.io/<repo>/
mkdir -p docs
mv tu-archivo.html docs/index.html
git add docs/index.html
git commit -m 'docs: publica guía .git/config en GitHub Pages'
git push origin main
